# Contexte projet — Revix

## En une phrase
Revix est une PWA d'apprentissage gamifiée pour étudiants du supérieur : upload de cours → fiches IA → quiz adaptatifs → révision espacée → progression gamifiée.

## Pour qui
Étudiants français (médecine, droit, ingénierie, etc.). Le cursus est configurable à l'inscription.

## Stack
- **Frontend** : React 18 + Vite 5 + TypeScript 5 + Tailwind CSS v3 + shadcn/ui
- **State serveur** : TanStack Query v5
- **Routing** : React Router v6
- **Backend** : Supabase (Lovable Cloud) — Postgres 15 + RLS + Edge Functions Deno
- **Auth** : Supabase Auth (email/password + Google OAuth)
- **IA** : Lovable AI Gateway (pas de clé API à gérer) — modèles Gemini 2.5 / GPT-5
- **Tests** : Vitest

## Design system (à respecter absolument)
- Style "néo-brutaliste papier" : bordures épaisses, ombres décalées, grain papier
- **Toutes les couleurs via tokens HSL** dans `src/index.css` et `tailwind.config.ts`
- **Jamais** de `text-white`, `bg-black`, `text-[#abc123]` dans les composants
- Effets signature : `.shadow-brutal`, `.paper-grain`, `.gradient-primary`
- Animations boutons : `hover:translate-x-[1px] hover:translate-y-[1px]`

## Architecture clé
```
src/
  pages/
    auth/          → Login, SignUp, Reset (AuthShell commun)
    app/           → Toutes les pages authentifiées
    Landing.tsx    → Landing publique
    Index.tsx      → Redirection selon auth
  components/revix/  → Composants métier
  components/ui/     → shadcn/ui (ne pas modifier)
  hooks/           → useAuth, useGamification, useSubscription, useUsage
  lib/             → coachContext.ts, gamification.ts, cosmetics.ts, pdf.ts
  integrations/supabase/
    client.ts      → ⚠️ AUTO-GÉNÉRÉ — ne JAMAIS éditer
    types.ts       → ⚠️ AUTO-GÉNÉRÉ — ne JAMAIS éditer
```

## Routes importantes
- `/` → Index (redirige)
- `/login`, `/signup`, `/reset-password` → Auth
- `/app` → Dashboard (hub)
- `/app/upload` → Upload + génération fiches IA
- `/app/fiches` → Liste cours
- `/app/fiches/:id` → Détail cours (fiches, quiz)
- `/app/quizz` → Hub quiz
- `/app/revision` → Révision espacée (SM-2)
- `/app/profil` → Profil + paramètres + suppression compte (RGPD)
- `/app/duel/:id` → Duels temps réel
- `/app/room/:id` → Salles d'étude collaboratives
- `/admin/ai-usage` → Admin (usage IA)

## Edge Functions (`supabase/functions/`)
| Fonction | Rôle |
|---|---|
| `extract-pdf` | OCR image → texte (quota fiche) |
| `generate-fiches` | Texte → fiches structurées IA |
| `generate-quiz` | Cours → QCM + questions ouvertes |
| `generate-planning` | Planning révision personnalisé IA |
| `grade-open` | Note réponse ouverte 0-10 |
| `coach-chat` | Chat coach IA (contexte complet) |
| `transcribe-voice` | Voix → texte |
| `oral-feedback` | Feedback oral IA |
| `room-explain` | Explication dans salle d'étude |

## Règles de sécurité
- **RLS activée sur TOUTES les tables**
- Rôles via `user_roles` + `has_role()` SECURITY DEFINER — jamais sur `profiles`
- Variables d'environnement `.env` auto-générées par Lovable

## Conventions
- Alias `@/` = `src/`
- Toasts : `import { toast } from "sonner"`
- Supabase client : `import { supabase } from "@/integrations/supabase/client"`
- Types DB : `import type { Database } from "@/integrations/supabase/types"`
- Edge functions : appel via `supabase.functions.invoke('nom', { body })`
- State serveur : TanStack Query, éviter `useEffect` + `fetch`
